DOM

  • 文档：一个页面就是一个文档，DOM中使用doucument来表示
  • 元素：页面中的所有标签都是元素，DOM中使用 element 表示
  • 节点：网页中的所有内容都是节点（标签，属性，文本，注释等），DOM中使用node表示

DOM 把以上内容都看做是对象

DOM重点核心

对于DOM操作，我们主要针对子元素的操作，主要有

  • 创建
  • 增
  • 删

  • 查
  • 属性操作
  • 事件操作

事件基础

JavaScript 使我们有能力创建动态页面，而事件是可以被 JavaScript 侦测到的行为。

触发— 响应机制。

网页中的每个元素都可以产生某些可以触发 JavaScript 的事件，例如，我们可以在用户点击某按钮时产生一个事件，然后去执行某些操作。

事件三要素

  1. 事件源
  2. 事件类型
  3. 事件处理程序

执行事件的步骤

  1. 获取事件源
  2. 注册事件(绑定事件)
  3. 添加事件处理程序(采取函数赋值形式)

节点概述

网页中的所有内容都是节点（标签、属性、文本、注释等），在DOM 中，节点使用 node 来表示。

HTML DOM 树中的所有节点均可通过 JavaScript 进行访问，所有 HTML 元素（节点）均可被修改，也可以创建或删除。

一般的，节点至少拥有nodeType（节点类型）、nodeName（节点名称）和nodeValue（节点值）这三个基本属性。

  • 元素节点：nodeType 为1
  • 属性节点：nodeType 为2
  • 文本节点：nodeType 为3(文本节点包括文字、空格、换行等)

结点层级

在实际开发中，节点操作主要操作的是元素节点

利用 DOM 树可以把节点划分为不同的层级关系，常见的是父子兄层级关系。

得到父级节点

node.parentNode

  • 得到离元素最近的父节点
  • 如果找不到父节点，则返回为空

得到子节点

node.childNodes

  • 得到所有子节点，包括文本节点，元素结点等…
  • 如果要得到元素结点则for遍历得到nodeTyoe = 1的结点

node.children

  • 获取所有子元素结点
  • 实际开发常用方法
  • 获取第一个或者最后一个子元素且无兼容问题

解决兼容性问题

node.firstChild

  • 获取第一个子节点
  • 包括文本节点，元素结点等

node.firstElementChild

  • 返回第一个子元素结点
  • IE9以上支持

node.lastChild

  • 获取最后一个子节点
  • 包括文本节点，元素结点

node.last ElementChild

  • 返回最后一个子元素结点
  • IE9以上支持

得到兄弟结点

node.nextSibling

node.previousSibling

  • 包含元素结点或者文本节点等…

node.nextElementSibling

node.previousElementSibling

  • 得到元素兄弟结点
  • 兼容性问题，IE9以上支持

解决兼容性问题

创建结点

document.creatElement(‘元素类型‘);

添加结点

node.appendChild(child);

  • 添加到指定父节点列表末尾
  • 即尾部追加元素

node.insertBefore(child, 指定元素);

  • 添加到指定父节点中指定元素前面

删除节点

node.removeChild(child);

复制结点(克隆结点)

node.cloneNode()

  • node.cloneNode()方法返回调用该方法的节点的一个副本。 也称为克隆节点/拷贝节点
  • 如果括号参数为空或者为 false ，则是浅拷贝，即只克隆复制节点本身，不克隆里面的子节点
  • 如果括号参数为 true ，则是深度拷贝，会复制节点本身以及里面所有的子节点
  • 默认为false

三种动态创建元素的区别

  • doucument.write()
  • element.innerHTML
  • document.createElement()

区别：

  • document.write() 是直接将内容写入页面的内容流，但是文档流执行完毕，则它会导致页面全部重绘
  • innerHTML 是将内容写入某个 DOM 节点，不会导致页面全部重绘，使用拼接字符串的方式
  • creatElement 创建多个元素效率更高（不要拼接字符串，采取数组形式拼接），结构稍微复杂

事件高级

注册事件

给元素添加事件，称为注册事件或者绑定事件。

注册事件有两种方式：传统方式和方法监听注册方式

传统注册方式 方法监听注册方式
利用 on 开头的事件 onclick w3c 标准推荐方式
<button onclick = "alert("hi")"></button> addEventListener()
btn.onclick = function() {} IE9 之前的 IE 不支持此方法，可使用 attachEvent() 代替
特点：注册事件的唯一性 特点：同一个元素同一个事件可以注册多个监听器
同一个元素同一个事件只能设置一个处理函数，最后注册的处理函数将会覆盖前面注册的处理函数 按注册顺序依次执行

事件监听方式

eventTarget.addEventListener(type, listener[, useCapture])
