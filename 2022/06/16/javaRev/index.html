  • Robust

  • Secure

  • Architecture neutral （体系结构中立）

  • Portable

  • High performance （高性能的）

  • Multithreaded

  • Dynamic language

Chapter 6 Methods: A Deeper Look

static Methods, static Fields

  • 静态方法只能调用相同类中的其他静态方法，其参数必须是常量、变量或者表达式
  • 静态成员被类的所有对象共享
  • 静态变量没有初始化时，编译器默认初始化

Why is method main declared static?

  • The JVM attempts to invoke the main method of the class you specify—when no objects of the class have been created.
  • Declaring main as static allows the JVM to invoke main without creating an instance of the class.

Three ways to call a method

  • 单独使用一个方法名来调用同一个类的另一个方法
  • 使用一个包含对象引用的变量，后跟一个点(.)和方法名来调用被引用对象的方法
  • 使用类名和点(.)调用类的静态方法

Chapter 8 Classes and Objects: A Deeper Look

keyword “this”

  • 在非static的方法中使用this表示引用当前对象的变量或者方法
  • 只能使用在非static的方法中
  • 用this显示或者隐式地引用

Default and No-Argument Constructors

  • 如果class中没有定义构造函数，则编译器自动生成一个无参的默认构造函数
    • zero for primitive numeric types, false for boolean values and null for references
  • 当class中定义构造函数以后，编译器不再生成默认构造函数
  • 构造函数不能被继承，但是可以由在子类中用suber关键字调用

Composition（has-a relationship）

  • 在类中创建另一个类的对象

enum

  • is-a relationship

Object Class

  • 所有类都直接或者间接继承于Object类

Chapter 10 Object-Oriented Programming: Polymorphism

Polymorphism

（多态）

  • program in the general
  • easily extensible 容易扩展
  • dynamic binding 动态绑定
  • aim a superclass reference at a subclass object 将父类引用指向子类对象

abstract class vs. interface

  • abstact class

    • 关键字abstact extends
    • 通常包含一个或者多个抽象方法
    • public abstract void draw(); //abstract方法示例
    • 构造函数和静态方法不可以声明为抽象方法
    • 一个类包含抽象方法必须声明为抽象类
    • 一个具体的抽象类子类必须包含所有抽象方法的声明
    • 抽象类不能创建实例，但是抽象类可以声明变量
    • 抽象类作为父类子类具有相关性

  • interface

    • 关键字interface implements
    • 接口提供的功能要求不相关的类实现一组通用方法
    • 接口定义并标准化了诸如人与系统等事物之间交互的方式
    • Java接口描述了可以在对象上调用的一组方法
    • 接口中声明的所有方法都是隐式public static方法
    • 所有字段都隐式地为public、static和final

instanceof

假设A为父类，B继承A，C继承B

只要判断为is-a关系即成立

Chapter 11 Exception Handling a deeper look

throw vs. throws

  • throw
    • 抛出一个异常
    • throw的操作数可以是派生自Throwable类的任何类
  • throws
    • 带有throws子句的方法被告知该方法可能抛出异常
    • 出现在方法参数列表之后和方法主体之前
    • 包含一个以逗号分隔的异常列表，其中列出了在发生各种问题时该方法将抛出的异常
    • 可以由方法体中的语句或从方法体中调用的方法抛出

finally

  • return, break, continue不会中断finally的执行，且一定会执行，除非System.exit(0);
  • 无论是否在相应的try块中抛出异常，finallt块都会执行
  • 通常放置在最后一个捕获块之后，用于重新分配资源

Throwable class

  • 其中Runtime Exception及其直接或间接子类为非受控异常（unchecked exceptions），其余均为受控异常（checked exceptions）

Checked exceptions vs. Unchecked exceptions

  • checked exception

    • 编译器对受控异常强制执行捕获声明要求

    • 编译器检查每个方法调用和方法声明，以确定该方法是否抛出检查过的异常

    • 如果是，编译器将验证所检查的异常是否被捕获或在throws子句中声明

    • Throws子句指定方法抛出的异常，这种异常通常不会在方法体中捕获

    • 如果不满足catch-or-declare要求，编译器将发出一条错误消息，指示必须捕获或声明异常

  • unchecked exception

    • 编译器不会检查代码以确定是否捕获或声明未检查的异常
    • 这些通常可以通过正确的编码来避免
    • 例如，如果方法在尝试执行除法之前确保分母不为零，则可以避免算术异常。不需要在方法的throws子句中列出未检查的异常

Rethrow an exception vs. Chained Exceptions

  • Rethrow an exception(重抛异常)

    • 重新抛出异常
    • 当catch块无法处理该异常或只能部分处理该异常时执行，将异常处理(或者异常处理的一部分)推迟到与外部try语句关联的另一个catch块
    • 当重新抛出发生时，下一个封闭的try块检测异常，该try块的catch块尝试处理

  • Chained Exceptions(链式异常)

    • 如果一个catch块抛出一个新的异常，原始异常的信息和堆栈跟踪将丢失
    • 链式异常使异常对象能够维护原始异常的完整堆栈跟踪信息

Declaring New Exception Types (cont.)

  • 构造新异常类需要包含四个构造函数
    • 不接受参数，并将默认错误消息String传递给父类构造函数
    • 接收定制的错误消息作为String，并将其传递给父类构造函数
    • 接收定制的错误消息为String和Throwable(用于链接异常)，并将它们都传递给父类构造函数
    • 另一个接收Throwable(用于链接异常)并将其传递给父类构造函数

Assertions

(断言)

  • 断言主要用于调试和识别应用程序中的逻辑错误
  • 可以指定方法执行前后的预期状态，前置条件和后置条件
  • 调用方法时，前提条件必须为真
  • 描述方法参数上的约束以及方法在开始执行之前对程序当前状态的任何其他期望
  • java -ea AssertTest 启用断言

Chapter 14 Strings, Characters and Regular Expressions

Class String

  • 不可更改的

  • 四个构造函数

class StringBuilder

  • 用于创建和操作动态字符串信息
  • 每个StringBuilder都能够存储由其容量指定的一些字符

class Character

  • 大多数Character方法都是为了方便处理单个字符值而设计的静态方法

Regular Expression

(正则表达式)

  • 正则表达式是一种特殊格式的String，用于描述在其他String中匹配字符的搜索模式
  • 对于验证输入和确保数据是特定格式的有用
  • 相关方法
    • String
      • String method matches
      • String method split
      • String method replaceAll | replaceFirst
    • Pattern
      • Pattern method matches
      • Pattern method compile
    • Matcher
  • 正则表达式的一个应用是简化编译器的构造
    • 通常，一个大而复杂的正则表达式被用来验证程序的语法
    • 如果程序代码与正则表达式不匹配，编译器就知道代码中存在语法错误
