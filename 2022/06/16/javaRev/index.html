Chapter 6 Methods: A Deeper Look

static Methods, static Fields

  • 静态方法只能调用相同类中的其他静态方法，其参数必须是常量、变量或者表达式
  • 静态成员被类的所有对象共享
  • 静态变量没有初始化时，编译器默认初始化

Why is method main declared static?

  • The JVM attempts to invoke the main method of the class you specify—when no objects of the class have been created.
  • Declaring main as static allows the JVM to invoke main without creating an instance of the class.

Three ways to call a method

  • 单独使用一个方法名来调用同一个类的另一个方法
  • 使用一个包含对象引用的变量，后跟一个点(.)和方法名来调用被引用对象的方法
  • 使用类名和点(.)调用类的静态方法

Chapter 8 Classes and Objects: A Deeper Look

keyword “this”

  • 在非static的方法中使用this表示引用当前对象的变量或者方法
  • 只能使用在非static的方法中

Default and No-Argument Constructors

  • 如果class中没有定义构造函数，则编译器自动生成一个无参的默认构造函数

  • 当class中定义构造函数以后，编译器不再生成默认构造函数
  • 构造函数不能被继承，但是可以由在子类中用suber关键字调用

Chapter 10 Object-Oriented Programming: Polymorphism

Polymorphism

（多态）

  • program in the general
  • easily extensible 容易扩展
  • dynamic binding 动态绑定
  • aim a superclass reference at a subclass object 将父类引用指向子类对象

abstract class vs. interface

  • abstact class

    • 关键字abstact extends
    • 通常包含一个或者多个抽象方法
    • public abstract void draw(); //abstract方法示例
    • 构造函数和静态方法不可以声明为抽象方法
    • 一个类包含抽象方法必须声明为抽象类
    • 一个具体的抽象类子类必须包含所有抽象方法的声明
    • 抽象类不能创建实例，但是抽象类可以声明变量
    • 抽象类作为父类子类具有相关性

  • interface

    • 关键字interface implements
    • 接口提供的功能要求不相关的类实现一组通用方法
    • 接口定义并标准化了诸如人与系统等事物之间交互的方式
    • Java接口描述了可以在对象上调用的一组方法
    • 接口中声明的所有方法都是隐式public static方法
    • 所有字段都隐式地为public、static和final

Chapter 11 Exception Handling a deeper look

throw vs. throws

  • throw
    • 抛出一个异常
    • throw的操作数可以是派生自Throwable类的任何类
  • throws
    • 带有throws子句的方法被告知该方法可能抛出异常
    • 出现在方法参数列表之后和方法主体之前
    • 包含一个以逗号分隔的异常列表，其中列出了在发生各种问题时该方法将抛出的异常
    • 可以由方法体中的语句或从方法体中调用的方法抛出

finally

  • return, break, continue不会中断finally的执行，且一定会执行，除非System.exit(0);
  • 无论是否在相应的try块中抛出异常，finallt块都会执行
  • 通常放置在最后一个捕获块之后，用于重新分配资源

Throwable class

  • 其中Runtime Exception及其直接或间接子类为非受控异常（unchecked exceptions），其余均为受控异常（checked exceptions）

Checked exceptions vs. Unchecked exceptions

  • checked exception

    • 编译器对受控异常强制执行捕获声明要求

    • 编译器检查每个方法调用和方法声明，以确定该方法是否抛出检查过的异常

    • 如果是，编译器将验证所检查的异常是否被捕获或在throws子句中声明

    • Throws子句指定方法抛出的异常，这种异常通常不会在方法体中捕获

    • 如果不满足catch-or-declare要求，编译器将发出一条错误消息，指示必须捕获或声明异常

  • unchecked exception

    • 编译器不会检查代码以确定是否捕获或声明未检查的异常
    • 这些通常可以通过正确的编码来避免
    • 例如，如果方法在尝试执行除法之前确保分母不为零，则可以避免算术异常。不需要在方法的throws子句中列出未检查的异常

Rethrow an exception vs. Chained Exceptions

  • Rethrow an exception(重抛异常)

    • 重新抛出异常
    • 当catch块无法处理该异常或只能部分处理该异常时执行，将异常处理(或者异常处理的一部分)推迟到与外部try语句关联的另一个catch块
    • 当重新抛出发生时，下一个封闭的try块检测异常，该try块的catch块尝试处理

  • Chained Exceptions(链式异常)

    • 如果一个catch块抛出一个新的异常，原始异常的信息和堆栈跟踪将丢失
    • 链式异常使异常对象能够维护原始异常的完整堆栈跟踪信息

Declaring New Exception Types (cont.)

  • 构造新异常类需要包含四个构造函数
    • 不接受参数，并将默认错误消息String传递给父类构造函数
    • 接收定制的错误消息作为String，并将其传递给父类构造函数
    • 接收定制的错误消息为String和Throwable(用于链接异常)，并将它们都传递给父类构造函数
    • 另一个接收Throwable(用于链接异常)并将其传递给父类构造函数

Assertions

(断言)

  • 断言主要用于调试和识别应用程序中的逻辑错误
  • 可以指定方法执行前后的预期状态，前置条件和后置条件
  • 调用方法时，前提条件必须为真
  • 描述方法参数上的约束以及方法在开始执行之前对程序当前状态的任何其他期望
  • java -ea AssertTest 启用断言

    • Pattern

Chapter 15 files, Input output Stream NIO and xmlserialization

Using NIO Classes and Interfaces to Get File and Directory Information

  • Path interface

    • 实现Path的类的对象表示文件或目录的位置

  • Paths class

    • Paths类提供用于获取表示文件或目录位置的Path对象的静态方法

  • Files class

    • Files类为常见的文件和目录操作提供静态方法

  • DirectoryStream interface

    • 实现此接口的类的对象使程序能够遍历目录的内容
  • Separator Characters(分隔符)
    • 在Windows计算机上，分隔符是一个反斜杠(\)；在Linux或macOS系统中，它是一个正斜杠(/)
    • Java在一个路径名中以相同的方式处理这两个字符

File I/O

  • 顺序存取

    • Formatter class
      • Formatter将格式化的字符串输出到指定的流
      • 带一个String参数的构造函数接收文件的名称，包括它的路径
      • 如果没有指定路径，JVM就假定文件在程序执行时所在的目录中
      • 如果该文件不存在，则创建该文件
      • 如果打开一个现有文件，它的内容将被截断
    • Scanner class
      • 为了从文件中顺序地检索数据，程序从文件的开头开始，并连续地读取所有数据，直到找到所需的信息
      • 在程序执行期间，可能需要对文件进行多次顺序处理(从文件的开头开始)
      • 该类不允许重新定位到文件的开头，程序必须关闭文件并重新打开它

  • 随机存取

    • RandomAccessFile
      • 读写流一体

Chapter 16 Generic Collections

  • Interface Collection
    • 包含添加、清除和比较集合中的对象的批量操作
    • 提供了一个返回Iterator对象的方法，该方法允许程序遍历集合并在迭代期间从集合中删除元素
    • 提供了对集合进行搜索、排序和执行其他操作的静态方法

  • Interface List

    • 列表索引是从零开始的
    • 提供了通过索引操作元素、操作指定范围的元素、搜索元素和获取访问元素的ListIterator的方法
    • 由几个类实现，包括ArrayList、LinkedList和Vector
      • 类ArrayList和Vector是List的可调整数组实现
      • ArrayList和Vector的主要区别在于Vector是默认同步的，而ArrayList不是
      • 在ArrayList或Vector的现有元素之间插入元素是低效的操作
      • LinkedList允许在集合中间高效地插入(或删除)元素
    • 因为它们只存储对对象的引用，所以向这些类的对象添加基元类型值时，会发生自动封装

  • Interface Queue

    • 继承接口Collection，提供了插入、删除和检查队列中的元素的附加操作
    • PriorityQueue按元素的自然顺序排序
      • 按照优先级顺序插入元素，这样优先级最高的元素(即最大的值)将成为从PriorityQueue中删除的第一个元素
      • 常见的PriorityQueue操作
        • offer根据优先级顺序在适当的位置插入一个元素
        • poll删除优先级队列中优先级最高的元素
        • peek获取优先级队列中优先级最高的元素的引用
        • clear该命令将删除优先队列中的所有元素
        • size获取队列中元素数量的大小
    • Sets (cont.)
      • 集合框架还包括SortedSet接口(它扩展了Set)，用于以有序顺序维护其元素的集合
  • Maps
    • 映射将键与值关联
    • Map中的键必须是唯一的，但关联的值不必是唯一的
    • 如果Map同时包含惟一键和惟一值，则称它实现了一对一映射
    • 如果只有键是唯一的，则Map实现了多对一映射—多个键可以映射到一个值
    • 实现接口Map的几个类中有三个是Hashtable、HashMap和TreeMap
    • 哈希表和HashMaps将元素存储在哈希表中，TreeMaps将元素存储在树中
    • 接口SortedMap扩展Map并按有序顺序维护其键—元素的自然顺序或Comparator指定的顺序
    • 类TreeMap实现了SortedMap

Chapter 17 Java SE 8 Lambdas and Streams

Intermediate operations vs.Terminal operation

  • Intermediate operations（中间操作）

    • use lazy evaluation
    • 产生一个新的流对象
    • map, filter, distinct, limit, sorted

  • Terminal operation（终止操作、终端操作）

    • use eager evaluation
    • 启动流管道的处理并产生结果
    • count, min, max, average, summary

lambda expression

（lambda表达式)

  • 表示匿名方法—实现函数式接口的速记符号，类似于匿名内部类
  • Lambda表达式的类型是lambda表达式实现的函数接口的类型
  • Lambda表达式可以在任何需要函数接口的地方使用
  • Lambda在其词法作用域中引用的任何局部变量都必须是final变量
  • Lambdas没有自己的作用域

functional interface

(函数式接口)

  • 只包含一个抽象方法的接口(也可能包含默认方法和静态方法)

  • Pure functions 纯函数

    • 只取决于它们的参数
    • 无副作用，不维持任何状态

  • referential transparency (RT, 引用透明性)

    • 如果表达式可以被其结果值替换而不改变程序的行为，那么表达式就是引用透明的
    • 无论表达式在程序中的何处使用，都必须为真

Chapter 20 Generic Classes and Methods

(泛型类)

Generic Classes

  • 如果操作由多个重载方法是相同的对于每个参数类型,重载的方法可以更简洁(简洁地)和方便编码使用一个通用的方法
  • 根据传递给泛型方法的参数类型，编译器适当地处理每个方法调用
  • 默认情况下，所有泛型类型都被替换为类型Object
  • 泛型方法可以重载
    • 类可以提供两个或多个泛型方法，它们指定相同的方法名称，但方法参数不同
    • 泛型方法也可以被非泛型方法重载
    • 当编译器遇到方法调用时，它会搜索与方法名和调用中指定的参数类型最精确匹配的方法声明

Raw Types

(原始类型)

Wildcards

(通配符)

  • 通配符能够指定作为参数化类型的父类型或子类型的方法参数、返回值、变量或字段

Chapter 23 Concurrency

  • Runnable interface

    • 创建多线程应用程序的首选方法
    • 声明方法run
    • 由实现Executor接口的对象执行

  • Executor interface

    • 声明方法execute
    • 创建并管理一组称为线程池的线程

  • ExecutorService interface

    • Executor的子接口，声明用于管理Executor生命周期的其他方法
    • 可以使用类Executors的静态方法创建
    • 方法shutdown在任务完成时结束线程

  • Executors class

    • 方法newFixedThreadPool创建一个由固定数量的线程组成的池
    • 方法newCachedThreadPool创建一个根据需要创建新线程的池

Thread Synchronization

  • 对共享对象的独占访问，在此期间，其他希望访问该对象的线程将保持等待
  • 在Java中使用锁实现
  • 实际上，线程同步只对共享的可变数据是必要的
  • 当跨线程共享不可变数据时，将相应的数据字段声明为final，以表明变量的值在初始化后不会更改
  • 关键字synchronized

Lock interface

  • 方法lock获取锁，强制互斥
  • 方法unlock释放锁

Producer/Consumer Relationship

  • 在生产者/消费者关系中，应用程序的生产者部分生成数据并将其存储在共享对象中，而应用程序的消费者部分从共享对象中读取数据
  • 生产者线程生成数据，并将其放在称为缓冲区的共享对象中
  • 消费者线程从缓冲区读取数据
  • 这种关系需要同步，以确保正确地生成和使用值
  • 生产者线程和消费者线程共享的缓冲区数据上的操作也是依赖于状态的—只有当缓冲区处于正确的状态时，操作才应该继续
  • 如果缓冲区处于非满状态，生产者可能产生; 如果缓冲区处于非空状态，则使用者可以进行消费
  • 同步生产者线程和消费者线程的一种方法是使用Java并发包中的类
    • Java包含类ArrayBlockingQueue-(来自Java .util.concurrent包)-一个完全实现的、线程安全的缓冲区类，它实现了接口BlockingQueue
    • 声明方法put和take，分别是Queue方法的阻塞等价物offer和poll
    • 方法将一个元素放置在BlockingQueue的末尾，如果队列已满则等待
    • 方法从BlockingQueue的头部移除一个元素，如果队列为空则等待
